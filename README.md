# Horizon Guard

Решение задачи регрессии для финтех-хакатона. По финансовым и поведенческим признакам клиента модель прогнозирует числовой `target` на нескольких временных горизонтах. Качество оценивается по **WMAE** - взвешенной средней абсолютной ошибке: чем она меньше, тем лучше.

## Как устроена модель

Итоговый пайплайн состоит из трёх уровней.

### Базовый ансамбль

В основе используются три детерминированные модели LightGBM с L1-функцией потерь. Их прогнозы объединяются взвешенным средним:

| Компонент           | Представление признаков                           | Итерации |  Вес |
| ------------------- | ------------------------------------------------- | -------: | ---: |
| `tuned_90`          | стабильные признаки без абсолютного номера месяца |     2300 | 0,36 |
| `wide_95`           | то же представление, более широкие деревья        |     1100 | 0,24 |
| `snapshot_tuned_90` | признаки с индексом месяца снимка                 |     2600 | 0,40 |

Основные модели имеют до 90–95 листьев и глубину до 9–10 уровней. Разные параметры обучения и подвыборки признаков делают компоненты менее похожими друг на друга и повышают устойчивость ансамбля.

### Source expert и коррекция ошибки

Поверх V4 применяется экспертный слой, который использует сильные денежные источники: зарплату, заявленный доход и средние выплаты. При достаточно надёжном сигнале он заменяет или смешивает базовый прогноз с оценкой по источнику.

Дополнительная residual-модель LightGBM учится предсказывать ошибку `target − прогноз V4` на out-of-time наблюдениях первого горизонта. Она использует 15 листьев, глубину 4 и 500 итераций. Экстремальные поправки ограничиваются квантилями ошибок, а итоговый прогноз - диапазоном от 20 000 до 1 500 000.

### Horizon Guard

Последний уровень отдельно защищает дальние горизонты от временного сдвига:

- для `h1` сохраняется прогноз V5;
- для `h2–h5` применяется дополнительная residual-коррекция;
- строки, в которых сработал надёжный source expert, остаются без этой коррекции.

Horizon Guard - компактный LightGBM на 418 признаках: 15 листьев, глубина 4, learning rate 0,025 и 400 итераций. Поправка умножается на коэффициент 1,25 и также ограничивается безопасным диапазоном.

## Признаки

Пайплайн строит признаки без использования `target` и обучает преобразования отдельно на доступном прошлом каждого временного фолда. Используются:

- исходные числовые признаки;
- категории, их частоты и признаки редких, неизвестных и пропущенных значений;
- характеристики текстового поля должности;
- разобранные коды регионов;
- время с последней активности;
- количество и структура пропусков по группам признаков;
- агрегаты зарплат и доходов, долговая нагрузка и финансовые отношения;
- signed-log преобразования денежных величин и их отклонения от базового прогноза.

Такой способ построения признаков вместе с expanding-window replay снижает риск утечки данных из будущих месяцев.

## Основные характеристики

| Параметр          | Значение                                            |
| ----------------- | --------------------------------------------------- |
| Базовый алгоритм  | Gradient Boosting Decision Trees, LightGBM          |
| Задача обучения   | L1-регрессия (`regression_l1`)                      |
| Метрика           | Weighted Mean Absolute Error, WMAE                  |
| Валидация         | последовательный expanding-window replay по месяцам |
| Горизонты         | `h1–h5`                                             |
| Диапазон прогноза | 20 000 – 1 500 000                                  |

## Состав проекта

- `pipeline.ipynb` - полный пайплайн подготовки данных, обучения моделей и построения прогноза;
- `submission.csv` - итоговый файл с предсказаниями.

## Запуск

1. Откройте `pipeline.ipynb` в Google Colab.
2. Загрузите в каталог `/content` файлы `train.csv`, `test.csv` и `sample_submission.csv`.
3. Выполните все ячейки ноутбука командой **Runtime → Run all**.
4. Скачайте сформированный файл `/content/submission.csv`.

Необходимые версии Python-библиотек устанавливаются в первой ячейке ноутбука. Основные зависимости: Python 3.12, LightGBM 4.6.0, pandas 3.0.4 и NumPy 2.5.0.
